タスク $ARGUMENTS の実装をサブエージェントに委譲してください。

## 手順

1. `doc/tasks/task_$ARGUMENTS.md` を読み、タスクの内容・完了条件・テスト内容を把握する
2. worktree が存在することを確認する: `gitworktree/feature-task$ARGUMENTS/`
   - 存在しない場合は「先に `/1_start-task $ARGUMENTS` を実行してください」と報告して中止
3. Task ツールで subagent_type="general-purpose" のサブエージェントを起動し、以下のプロンプトを渡す:

---

あなたはタスク $ARGUMENTS の実装担当です。

**最初に以下のファイルを必ず読んでください:**
- `CLAUDE.md`（プロジェクト全体のルール・アーキテクチャ）
- `doc/tasks/task_$ARGUMENTS.md`（このタスクの詳細・完了条件）
- `doc/design.md`（設計方針・ディレクトリ構成）

**作業ディレクトリ:** `gitworktree/feature-task$ARGUMENTS/`
このディレクトリ内のファイルのみを編集してください。リポジトリルートのファイルは編集しないでください。

**実装手順:**
1. 上記ファイルを読んでタスクの要件を理解する
2. 必要に応じて既存コードを読み、現在の実装状況を把握する
3. タスクの内容に従って実装する
4. テスト項目がある場合はテストコードも作成する
5. `npm run build` でビルドエラーがないことを確認する（プロジェクトセットアップ済みの場合）
6. テストがある場合は `npx vitest run` でテストが通ることを確認する
7. 変更内容をコミットする。コミットメッセージには `task$ARGUMENTS:` プレフィックスを付ける（例: `task001: add vite project template`）
8. 実装内容のサマリと、完了条件の達成状況を報告する

---

4. サブエージェントの結果を受け取り、ユーザに以下を報告する:
   - 実装内容のサマリ
   - 完了条件の達成状況
   - 次のステップ（`/3_finish-task $ARGUMENTS` の案内）
