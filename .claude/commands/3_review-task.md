タスク $ARGUMENTS の実装をレビューしてください。

## 手順

1. `doc/tasks/task_$ARGUMENTS.md` を読み、タスクの内容・完了条件・テスト内容を把握する
2. worktree が存在することを確認する: `gitworktree/feature-task$ARGUMENTS/`
   - 存在しない場合は中止
3. Task ツールで subagent_type="general-purpose" のサブエージェントを起動し、以下のプロンプトを渡す:

---

あなたはタスク $ARGUMENTS のコードレビュー担当です。

**最初に以下のファイルを必ず読んでください:**
- `CLAUDE.md`（プロジェクト全体のルール・アーキテクチャ）
- `doc/tasks/task_$ARGUMENTS.md`（このタスクの詳細・完了条件）
- `doc/design.md`（設計方針・ディレクトリ構成）

**作業ディレクトリ:** `gitworktree/feature-task$ARGUMENTS/`

**レビュー手順:**

1. `git diff develop...HEAD` で差分を確認し、変更されたファイルの全体像を把握する
2. 変更された各ファイルを読み、以下の観点でレビューする:

   **完了条件の充足:**
   - `doc/tasks/task_$ARGUMENTS.md` の完了条件チェックリストを1つずつ検証する
   - 未達の項目があれば明記する

   **コード品質:**
   - TypeScript の型が適切に使われているか
   - `doc/design.md` のディレクトリ構成・命名規約に従っているか
   - 不要なコード・コメント・デバッグ用コードが残っていないか
   - エッジケースの考慮漏れがないか

   **テスト:**
   - テスト項目がタスクに記載されている場合、テストコードが存在するか
   - `npx vitest run` でテストが全て通るか（テストがある場合）

   **ビルド:**
   - `npm run build` でビルドエラーがないか

3. レビュー結果を以下のフォーマットで報告する:

   ```
   ## レビュー結果: task$ARGUMENTS

   ### 判定: OK / NG

   ### 完了条件チェック
   - [x] 条件1
   - [x] 条件2
   - [ ] 条件3（未達: 理由）

   ### 指摘事項
   - **[重要]** 〜〜（マージ前に修正必須）
   - **[提案]** 〜〜（改善提案、今回は任意）

   ### テスト結果
   - vitest: PASS / FAIL（詳細）
   - build: PASS / FAIL（詳細）
   ```

**重要:** レビューではコードの修正を行わないでください。問題の指摘と報告のみ行ってください。

---

4. サブエージェントの結果を受け取り、ユーザに報告する:
   - **OK の場合**: `/4_finish-task $ARGUMENTS` でマージ可能と案内する
   - **NG の場合**: 指摘事項を報告し、`/2_impl-task $ARGUMENTS` で修正が必要と案内する
