# Task 003: `public/players.json` の仮データ作成

## フェーズ

1. プロジェクト基盤

## 依存タスク

* Task 001

## 内容

シミュレーションに必要な選手データの仮データを作成する。最低10選手×2チーム。

### やること

* `public/players.json` を作成
* 各選手に以下のフィールドを持たせる:
  * `id`: ユニークID
  * `name`: 選手名
  * `team`: チーム名
  * `pa`: 打席数 (Plate Appearances)
  * `single` (1B), `double` (2B), `triple` (3B), `hr`: 各安打数
  * `bb`: 四球数
  * `hbp`: 死球数
  * `so`: 三振数
* 2チーム（例: "チームA", "チームB"）各10選手以上
* 数値はリアルな範囲にする（PA: 300〜600程度、打率.200〜.300程度の分布）

## 完了条件

- [ ] `public/players.json` が存在する
- [ ] 2チーム以上、各10選手以上のデータが含まれる
- [ ] 全選手に必要な全フィールドが存在する
- [ ] `1B + 2B + 3B + HR + BB + HBP + SO <= PA` が全選手で成立する
- [ ] JSONとしてパース可能

## テスト

* `cat public/players.json | jq .` でパースエラーがないこと
* 各選手の数値整合性を目視確認
