# Task 017: `SimulationPanel` 実行ボタン・進捗表示

## フェーズ

5. Worker化と画面2

## 依存タスク

* Task 008
* Task 016

## 内容

シミュレーションの設定・実行・結果表示を行う画面2のメインパネルを実装する。

### やること

* `src/components/SimulationPanel.svelte` を作成
* `src/stores/ui.ts` に `simConfigStore`（N, seed）と `simResultStore` を追加
* UI要素:
  * 試行回数 N の入力（1,000〜100,000、デフォルト10,000）
  * seed 入力（任意、空ならランダム）
  * 「シミュレーション実行」ボタン
  * 実行中のローディング表示（簡易: spinner やテキスト）
  * 結果サマリ表示: 平均得点、中央値、P10、P90
* バリデーション:
  * 打線に未設定枠がある場合 → ボタン disabled + 理由表示
  * PA=0 の選手がいる場合 → ボタン disabled + 理由表示
* タブ切替 UI の実装（画面1: 打線作成、画面2: シミュレーション）

## 完了条件

- [ ] N と seed の入力UIが存在する
- [ ] 「実行」ボタンでシミュレーションが実行され、結果が表示される
- [ ] 実行中にローディング表示が出る
- [ ] 打線未完成時にボタンが disabled で理由が表示される
- [ ] タブ切替で画面1/画面2を行き来できる

## テスト

* 手動確認: 9枠すべて埋めて「実行」→ 結果（mean等）が表示される
* 手動確認: 枠が空のまま → ボタン disabled + 「打線が未完成です」等
* 手動確認: タブ切替で画面遷移が正しく動作する
