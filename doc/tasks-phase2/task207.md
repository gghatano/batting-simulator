# Task 207: 打線パネルのビジュアル改善

## フェーズ

Phase 2 - UIデザインブラッシュアップ

## 依存タスク

* Task 201
* Task 204

## 内容

打線パネル（LineupPanel + LineupSlot）のビジュアルを改善し、操作性を向上させる。

### やること

* `LineupSlot.svelte`:
  * 打順番号を大きく表示（1.8rem、テーマカラー背景の円形バッジ）
  * 選択中の枠を明確にハイライト:
    * 左ボーダーに 4px テーマカラーの太線
    * 背景色をより濃く
  * 操作ボタン（↑↓×）のサイズを拡大（最低 32x32px）
  * 空枠のプレースホルダを破線ボーダー + 薄い背景に変更
  * ハードコード色をトークンに置換
* `LineupPanel.svelte`:
  * ボタン群のスタイルをトークンベースに統一
  * 「シミュレーション開始」ボタンをより目立たせる（テーマカラー、大きめ）
  * セクション間の余白をトークンで統一
* `LineupSummary.svelte`:
  * ハードコード色をトークンに置換
  * 全体的なスタイルをトークンベースに統一

## 完了条件

- [ ] 打順番号が大きく見やすい
- [ ] 選択中の枠が一目で分かる（左ボーダー太線 + 濃い背景）
- [ ] 操作ボタンが 32x32px 以上
- [ ] 空枠が破線ボーダーで「ここに入る」と直感的に分かる
- [ ] ハードコード色がトークンに置換されている
- [ ] `npm run build` が成功する
