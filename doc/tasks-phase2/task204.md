# Task 204: カラースキームの確定・適用

## フェーズ

Phase 2 - UIデザインブラッシュアップ

## 依存タスク

* Task 201（トークン定義を更新するため）

## 内容

野球アプリらしいテーマカラーを選定し、トークンの色定義を確定、主要UIに適用する。

### やること

* テーマカラーの選定（推奨: ダークネイビー `#1e3a5f` + ゴールドアクセント `#d4a843`、または グリーン系）
* `src/styles/tokens.css` のカラートークンを確定値に更新:
  * primary 系列: テーマカラーの明度バリエーション
  * danger 系列: `#dc2626` ベースで統一
  * neutral 系列: 灰色の段階を整理
  * 背景色: メイン `#f5f6f8`, サーフェス `#ffffff`, ミュート `#eef1f5`
* 適用箇所:
  * App.svelte のタブ: アクティブ色をテーマカラーに
  * ボタン類: primary ボタンをテーマカラーに統一
  * LineupSlot: 選択中の色をテーマカラーに
* WCAG AA コントラスト比 4.5:1 以上を確保（テキスト色 vs 背景色）

## 完了条件

- [ ] テーマカラーが tokens.css に反映されている
- [ ] タブ・ボタン・選択状態がテーマカラーで統一されている
- [ ] `#dc2626` 以外の赤（`#c00`, `#fca5a5` 等）がトークンに統一されている
- [ ] 主要テキストのコントラスト比が 4.5:1 以上
- [ ] `npm run build` が成功する

## テスト

* 手動確認: アプリ全体のカラーが統一されている
* 手動確認: テーマカラーで野球アプリらしい雰囲気が出ている
* ツール確認: Chrome DevTools の Accessibility パネルでコントラスト比チェック
