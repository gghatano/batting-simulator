# Task 104: シミュレーション画面の2カラムレイアウト（打線固定表示）

## フェーズ

Phase 1 - UI改善

## 依存タスク

なし

## 内容

シミュレーション画面（画面2）を左右2カラムにし、右側に打線を常時固定表示する。

### やること

* `SimulationPanel.svelte`（または `App.svelte` のシミュレーションタブ部分）を2カラムレイアウトに変更
  * 左: シミュレーション結果（既存の内容）
  * 右: 打線表示（`LineupPanel` の読み取り専用版、または打線サマリコンポーネント）
* 右カラムを `position: sticky; top: 1rem` で固定
* 右カラムに以下を表示:
  * 現在の打線（1〜9番の選手名）
  * 「打線作成へ戻る」ボタン（`activeTab` を `'lineup'` に切替）
  * 「再シミュレーション」ボタン（任意）
* `LineupPanel` を画面1/画面2で共通利用できるようにする（表示モードを prop で切替、または別コンポーネント）

## 完了条件

- [ ] シミュレーション画面が左右2カラムで表示される
- [ ] 右カラムに現在の打線（1〜9番）が表示される
- [ ] 右カラムがスクロールしても固定される（sticky）
- [ ] 「打線作成へ戻る」ボタンで画面1に遷移する
- [ ] `npm run build` が成功する

* 手動確認: 「打線作成へ戻る」ボタンで画面1に遷移する
* 手動確認: 打線の内容が画面1で設定したものと一致する
