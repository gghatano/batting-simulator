# Task 105: シミュレーション画面 即時サマリ（100試合自動実行）

## フェーズ

Phase 1 - UI改善

## 依存タスク

* Task 104（2カラムレイアウト完成後に左カラムに配置するため）

## 内容

シミュレーション画面に遷移した直後に、打線で100試合を自動実行し、結果サマリを最上段に表示する。

### やること

* `src/components/SimulationSummary.svelte` を新規作成
* シミュレーション画面への遷移時（`activeTab === 'simulation'` になった時）に自動で100試合を実行
  * Web Worker 経由（既存の `runSimulation` を使用、N=100）
* 表示項目:
  * 平均得点
  * 中央値
  * P10 / P90
  * 最大 / 最小
  * 得点分布（表形式: 0〜n点）
* UX:
  * 計算中: スピナー + 「100試合を実行中...」表示
  * 完了後: サマリを表示
* 打線が変更された場合（画面1に戻って変更して再遷移）は再実行する
* `SimulationPanel.svelte` の最上段に `SimulationSummary` を配置

## 完了条件

- [ ] シミュレーション画面遷移時に自動で100試合が実行される
- [ ] 計算中にスピナー/ローディング表示が出る
- [ ] 完了後に平均得点、中央値、P10/P90、得点分布が表示される
- [ ] 打線変更後の再遷移で結果が更新される
- [ ] UIがフリーズしない（Worker実行）
- [ ] `npm run build` が成功する

## テスト

* Vitest: seed 固定で100試合の結果が再現可能
* 手動確認: 画面遷移 → ローディング → サマリ表示の流れ
* 手動確認: 打線変更 → 再遷移 → 結果が変わる
