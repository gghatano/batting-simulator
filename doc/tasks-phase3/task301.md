# Task 301: 選手カードのコンパクト化

## フェーズ

Phase 3 - UX改善・機能追加

## 依存タスク

* なし

## 背景

選手カードが縦に大きすぎて、一度に表示できる選手数が少ない。追加ボタンも大きすぎる。カード上の情報を厳選し、コンパクトにして一覧性を高める。

## 内容

### やること

* `PlayerCard.svelte` の表示情報を整理:
  * 名前を大きめに表示（`--font-base` 以上、`font-weight: 700`）
  * 成績表示を **打率** と **HR率** の2つのみに絞る
    * 打率 = `(single + double + triple + hr) / pa`
    * HR率 = `hr / pa`
  * PA, 四死率, K率 の表示を削除
* 追加ボタンを小さくする:
  * `width: 100%` → 右寄せの小さなボタンに変更
  * `min-height: 32px` → `min-height: 28px` 程度
  * または「+」アイコンだけのコンパクトボタン
* カード全体の高さを削減:
  * padding を縮小
  * gap を詰める
  * card-footer の border-top を削除して省スペース化
* `PlayerList.svelte` のグリッド `minmax` を調整（カードが小さくなるので `minmax(150px, 1fr)` 程度）

## 完了条件

- [ ] カード上の成績表示が打率とHR率のみ
- [ ] 名前が目立つサイズで表示されている
- [ ] 追加ボタンが以前より小さい
- [ ] カード高さが削減され、一画面に表示できる選手数が増えている
- [ ] `npm run build` が成功する

## テスト

* 手動確認: カード上に打率・HR率のみ表示
* 手動確認: 画面あたりの選手カード表示数が以前より多い
